# EduRealtime

A live virtual classroom platform built on Cloudflare's infrastructure. Supports real-time video sessions, an AI teaching assistant (EduBot), live transcription, breakout rooms, and post-session AI summaries.

---

## What's included

- Video/audio classrooms with instructor and student roles
- AI teaching assistant with streaming responses and optional voice output
- Live transcription with real-time captions
- Breakout rooms (switch in-place, no new tab)
- Guest join via 6-character code — no account needed for students
- Post-session summaries generated by Workers AI
- Deploys to Cloudflare Workers + D1

---

## Prerequisites

Before you start, you need:

- [Node.js](https://nodejs.org) v18 or later
- A [Cloudflare account](https://dash.cloudflare.com/sign-up) (free tier works)
- Cloudflare **RealtimeKit** enabled on your account (request access at [developers.cloudflare.com/realtime](https://developers.cloudflare.com/realtime/))

## 1. Clone and install

```bash
git clone https://github.com/kiishi-19/edu-realtime.git
cd edu-realtime
npm install
```

---

## 2. Create a Cloudflare D1 database

D1 is Cloudflare's serverless SQL database. Create one for this project:

```bash
npx wrangler d1 create edu-realtime
```

Copy the `database_id` from the output and paste it into `wrangler.toml`:

```toml
[[d1_databases]]
binding = "DB"
database_name = "edu-realtime"
database_id = "PASTE_YOUR_DATABASE_ID_HERE"
```

---

## 3. Set up Cloudflare RealtimeKit

1. Go to the [Cloudflare dashboard](https://dash.cloudflare.com) → **Realtime** → **Kit**
2. Create a new RTK application
3. Copy the **App ID** — you'll need it in the next step
4. Inside your RTK app, go to **Presets** and confirm at least two presets exist (one for hosts, one for participants). Cloudflare creates defaults like `group-call-host` and `group-call-participant` automatically

---

## 4. Configure environment variables

Copy the example file and fill in your values:

```bash
cp .env.example .env.local
```

Open `.env.local` and set:

```env
# Your Cloudflare account ID
# Found at: dash.cloudflare.com → right sidebar
CLOUDFLARE_ACCOUNT_ID=your_account_id_here

# API token with "Edit Cloudflare Workers" permissions
# Create at: dash.cloudflare.com/profile/api-tokens
# Use the "Edit Cloudflare Workers" template
CLOUDFLARE_API_TOKEN=your_api_token_here

# RealtimeKit App ID from step 3
CLOUDFLARE_RTK_APP_ID=your_rtk_app_id_here

# Any long random string — used to sign auth cookies
# Generate one with: openssl rand -hex 32
JWT_SECRET=change-me-to-a-long-random-string

# Local dev URL (leave as-is for development)
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### Optional — ElevenLabs voice (TTS)

EduBot supports text-to-speech. Without a key it falls back to browser speech, but ElevenLabs gives significantly better audio quality:

```env
ELEVENLABS_API_KEY=your_elevenlabs_key
ELEVENLABS_VOICE_ID=21m00Tcm4TlvDq8ikWAM   # optional, defaults to Rachel
```

Get a free key at [elevenlabs.io](https://elevenlabs.io) (10,000 characters/month free).

### Optional — pin RTK preset names

The app auto-detects your RTK presets, but you can pin them explicitly if needed:

```env
RTK_INSTRUCTOR_PRESET=group-call-host
RTK_STUDENT_PRESET=group-call-participant
```

To see what presets your RTK app has, start the dev server and visit `GET /api/debug/rtk`.

## 5. Run locally

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). The local dev server uses SQLite (no D1 setup needed for development).

**First time setup:**
1. Go to `/register` and create an instructor account
2. Create a session from the dashboard
3. Share the 6-character join code with students — they can join at `/join` without an account

## 7. Deploy to Cloudflare Workers

### 7a. Create a Cloudflare API token for deployment

1. Go to [dash.cloudflare.com/profile/api-tokens](https://dash.cloudflare.com/profile/api-tokens)
2. Click **Create Token**
3. Select the **"Edit Cloudflare Workers"** template
4. Set Account Resources to your account
5. Create the token and copy it

Set the token in your terminal before deploying (do not paste it into `.env.local` for production — use Wrangler secrets instead):

```bash
export CLOUDFLARE_API_TOKEN=your_token_here
```

### 7b. Set production secrets

Wrangler secrets are encrypted and injected at runtime — use them for sensitive values:

```bash
npx wrangler secret put JWT_SECRET
npx wrangler secret put CLOUDFLARE_RTK_APP_ID
npx wrangler secret put ELEVENLABS_API_KEY   # if using ElevenLabs
```

Non-secret variables like `NEXT_PUBLIC_APP_URL` go in `wrangler.toml` under `[vars]`:

```toml
[vars]
NEXT_PUBLIC_APP_URL = "https://edu-realtime.YOUR_SUBDOMAIN.workers.dev"
```

### 7c. Run the D1 migration

Apply the database schema to your production D1 database:

```bash
npx wrangler d1 execute edu-realtime --file=migrations/001_schema.sql --remote
```

### 7d. Deploy

```bash
npm run deploy
```

Your app will be live at `https://edu-realtime.YOUR_SUBDOMAIN.workers.dev`.

## Environment variable reference

| Variable | Required | Description |
|---|---|---|
| `CLOUDFLARE_ACCOUNT_ID` | Yes | Your Cloudflare account ID |
| `CLOUDFLARE_API_TOKEN` | Yes | API token with Workers edit permissions |
| `CLOUDFLARE_RTK_APP_ID` | Yes | RealtimeKit application ID |
| `JWT_SECRET` | Yes | Secret for signing auth cookies |
| `NEXT_PUBLIC_APP_URL` | Yes | Full URL of the deployed app |
| `ELEVENLABS_API_KEY` | No | Enables high-quality TTS for EduBot |
| `ELEVENLABS_VOICE_ID` | No | ElevenLabs voice ID (defaults to Rachel) |
| `RTK_INSTRUCTOR_PRESET` | No | Pins the RTK preset used for instructors |
| `RTK_STUDENT_PRESET` | No | Pins the RTK preset used for students |

## Troubleshooting

**"No presets found in your RTK app"**
Go to your RTK app in the Cloudflare dashboard → Presets and confirm at least one preset exists. Visit `/api/debug/rtk` (dev only) to inspect what the app can see.

**Wrangler deploy fails with authentication error**
Make sure your `CLOUDFLARE_API_TOKEN` was created from the "Edit Cloudflare Workers" template and has `Workers Scripts: Edit` and `User Details: Read` permissions.

**STT (speech-to-text) not working**
Browser speech recognition only works in Chrome and Edge. Safari and Firefox are not supported.

**Students can't join**
Check that the session status is `active` (not `ended`) and that the join code is correct. Instructors need to be in the room first for RTK to accept new participants.
